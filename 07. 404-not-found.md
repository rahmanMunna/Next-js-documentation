# Custom 404 Not Found Page in Next.js

This documentation explains how to create and customize a **404 Not Found** page in Next.js, including handling dynamic routes and displaying parameters when a resource is not found.

- Default export a function to render your custom 404 page.

```tsx

  return <h1>404 - Page Not Found</h1>;
}
```

## ðŸ“‚ Specific Not Found Page for Dynamic Routes
### Folder structre:
```
app/
  category/
      [cId]/
        product/
            [pId]/
```
### Logic for Product ID
- If the product ID (pId) is greater than 1000, show a custom 404 page.
- Create a file named not-found.tsx inside the [pId] route.
- Use the notFound function from next/navigation.

```
import { notFound } from "next/navigation";

export default function ProductPage({ params }) {
  const { pId } = params;

  if (parseInt(pId) > 1000) {
    notFound(); // triggers the custom 404 page
  }

  return <h1>Product {pId}</h1>;
}
```
- It will show out specific route not-found page.
## ðŸ”Ž Displaying cId and pId in the Not Found Pag
- We can extract cId and pId with params.
- But in `not-found` page we can use params.
- So we can use `usePathname` hooks.
- For that we need to page this component as client component
- include `"use client"` at top of the file
```
"use client"

import { usePathname } from "next/navigation"

export default function NotFound() {
    const path = usePathname()
    const cId = path.split("/")[2];
    const pId = path.split("/")[4];
    return <h1>404 Not Found category {cId} product {pId}</h1>
}
```
