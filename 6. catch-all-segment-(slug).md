# üìò Catch-All Segments Using Slug in Next.js
## What is a Slug?
### A slug is simply a dynamic route parameter, usually used as a human-readable identifier in URLs.

# üö© Problem: Deep Nested Routes
- suppose Url : docs/features/concept
  - For this url we need 3 folder nested one another docs ‚Üí features ‚Üí concept
    
  - But if we want :
      - docs/features1/concept1
      - docs/features2/concept2
      - ....
  - Suppose we have 20 features,20 concept
  - Now we need 20 * 20 = 40 folder
    - ## ‚úÖ Solution: Dynamic Routes:
      - `docs/[conceptId]/[conceptId]`
    - It down in only 2 folder from 40
     - ## ‚ö†Ô∏è But There are still a catch
        - What if we introduce a deeper nested structure?
        - What if we need an example route : `docs/[conceptId]/[conceptId]/example`
        - We need to do another nesting routing
        - So here comes slug , that can catch all segments
# üöÄ Catch-All Route With [...slug]
- A catch-all slug can capture every part of the URL in a single variable.

Folder structure : 
```
app/
  docs/
    [...slug]/
      page.tsx
```
### This route can match ALL of the following:
  - /docs/feature1
  - /docs/feature1/concept1
  - /docs/feature1/concept1/example
  - /docs/a/b/c/d/e/... (any depth)
```page.tsx
export default async function Docs({ params }: { params: Promise<{ slug: string[] }> }) {
    const { slug } = await params

    if (slug?.length == 1) {
        return <h1>This is docs {slug[0]}</h1> // docs/[conceptId]
    }
    if (slug?.length == 2) {
        return <h1>This is docs {slug[0]} and {slug[1]}</h1> // docs/[conceptId]/[conceptId]
    }
    if (slug?.length == 3) {
        return <h1>This is docs {slug[0]} and {slug[1]} and {slug[2]}</h1> // docs/[conceptId]/[conceptId]/example
    }
    return <h1>This is docs</h1>
}
```
- Slug will be an array of string
## ‚ùó Important Issue
- If We request for : `localhost:3000/docs`
- It will give `404 not found`
- Because a catch-all route ([...slug]) only matches one or more segments ‚Äî not zero.
## ‚úÖ Fix: 
```
app/
  docs/
    page.tsx        <-- handles /docs
    [...slug]/
      page.tsx      <-- handles /docs/*
```
- We can create another `page.tsx` inside docs route
